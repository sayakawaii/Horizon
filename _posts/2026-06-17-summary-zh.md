<summary><span class="hz-item-title">RFC 10008 定义新的 HTTP QUERY 方法</span> <span class="hz-item-score">⭐️ 8.0/10</span></summary>

RFC 10008 引入了 HTTP QUERY 方法，这是一种安全且幂等的替代方案，用于带请求体的 GET 请求，使得带负载的请求可以被缓存并安全地重复执行。 该标准将常见的变通做法（带请求体的 GET）规范化，解决了互操作性问题，并支持对复杂查询进行缓存，从而提升 API 的效率和可靠性。 QUERY 类似于 POST，但安全且幂等，意味着可以自动重试而不会产生副作用。请求体是缓存键的一部分，这可能会使缓存策略变得复杂。

🔗 [来源](https://www.rfc-editor.org/info/rfc10008/)

hackernews · schappim · 6月17日 10:51 · [社区讨论](https://news.ycombinator.com/item?id=48568502)

**背景**: HTTP 方法如 GET 是安全且幂等的，但不支持请求体；而 POST 支持请求体，但并非幂等。开发者经常使用带请求体的 GET，导致互操作性问题。新的 QUERY 方法填补了这一空白。
