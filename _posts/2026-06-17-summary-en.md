<summary><span class="hz-item-title">RFC 10008 Defines New HTTP QUERY Method</span> <span class="hz-item-score">⭐️ 8.0/10</span></summary>

RFC 10008 introduces the HTTP QUERY method, a safe and idempotent alternative to GET with a body, enabling requests with payloads to be cached and safely repeated. This standardizes a common workaround (GET with body) that had interoperability issues, and enables caching of complex queries, improving API efficiency and reliability. QUERY is similar to POST but is safe and idempotent, meaning it can be automatically retried without side effects. The request body is part of the cache key, which may complicate caching strategies.

🔗 [Source](https://www.rfc-editor.org/info/rfc10008/)

hackernews · schappim · Jun 17, 10:51 · [Discussion](https://news.ycombinator.com/item?id=48568502)

**Background**: HTTP methods like GET are safe and idempotent but do not support a request body, while POST supports a body but is not idempotent. Developers often used GET with a body, causing interoperability issues. The new QUERY method fills this gap.
